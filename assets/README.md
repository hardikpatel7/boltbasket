# Assets

Diagrams, screenshots, and images for articles.

## Conventions

- File naming: `<article-slug>-<asset-name>.<ext>` — e.g., `meet-boltbasket-stack-diagram.png`
- Diagrams: prefer simple block diagrams (Excalidraw, Mermaid, Figma). Avoid AI-generated illustrations.
- Screenshots of query results: take from the BoltBasket Supabase once Phase 4 is live.
- File sizes: compress before committing. Medium handles large images poorly on mobile.

## Diagram style guide

- Boxes for systems/components.
- Arrows for data flow, labeled with what flows.
- Annotations for what's painful (a small "💥" or "[broken]" callout where the imperfections hit).
- BoltBasket brand color (decide later) for accents.
- Don't over-design. Boxes-and-arrows beat polished isometric illustrations for technical credibility.
